Adresse IP

Les adresses IP ont deux versions ou normes distinctes.
L’adresse IPv4 (Internet Protocol version 4) est l’adresse la plus ancienne des deux, 
qui peut accueillir jusqu’à 4 milliards d’adresses IP et est attribuée à tous les ordinateurs.
La version 6 du protocole Internet (IPv6) la plus récente peut accueillir des milliards d’adresses IP, 
ce qui représente la nouvelle génération d’appareils en plus des ordinateurs. Il existe également plusieurs types d’adresses IP, 
y compris les adresses IP publiques, privées,  statiques et dynamiques.

IPv4 est la quatrième version de l’IP. 
Il s’agit de l’un des protocoles fondamentaux des méthodes basées sur des normes utilisées pour interconnecter Internet et d’autres réseaux. 
Le protocole a été déployé pour la première fois sur l’Atlantic Packet Satellite Network (SATNET), 
qui était un réseau satellite qui constituait un segment des premières étapes d’Internet, en 1982. 
Il est toujours utilisé pour acheminer la plupart du trafic Internet malgré l’existence d’IPv6.

IPv4 est actuellement attribué à tous les ordinateurs. 
Une adresse IPv4 utilise des numéros binaires de 32 bits pour former une adresse IP unique. 
Il prend le format de quatre ensembles de chiffres, chacun allant de 0 à 255 et représente un nombre binaire à huit chiffres, 
séparé par un point de point.

=================================================================

socket :

Creer la porte d'entree pour permettre
la communication du serveur avec le client
renvoie un fd en cas de reussite ou -1 en cas d'echec.

=================================================================

setsockopt : 

fais en sorte de pouvoir reutiliser un port
Par exemple si je lance mon programme, que je le quitte et que je le relance, sans
cette fonction j'aurais une erreur du genre "port already in use"

int setsockopt(int socket, int level, int option_name,
       const void *option_value, socklen_t option_len);

retourne 0 en cas de succes ou -1 en cas d'erreur

=================================================================

bind :

prend mon socket et l'attache a une adresse ip locale et a un port specifique
verifie si le port est deja utilise
utile pour l'initialisation du serveur qui a besoin d'un port specifique

0 = succes | -1 = erreur

=================================================================

listen :

Techniquement, la fonction listen() transforme ton socket 
(qui est bidirectionnel par défaut) en un socket passif. 
Cela indique au système d'exploitation 
que ce socket ne servira jamais à envoyer des données de son propre chef,
mais qu'il servira uniquement à attendre et accepter des connexions entrantes.

0 = succes | -1 = erreur

=================================================================

poll :

La raison pour laquelle le sujet nous conseille d'utiliser un equivalent
est parce que poll() parcourt toute la liste de client pour voir s'il y a du nouveau
sur 100 clients la difference ne se fait pas sentir, sur 10 000, c'est tres lent.

epoll() est un bon equivalent sur linux.

=================================================================

fcntl (file control) :

realise une operation sur le file descriptor fd
dans le projet on change la propriete d'un fd qui est bloquant par defaut en non bloquant.
c'est pour eviter que le serveur se fige quand il attend qu'un client fasse quelque chose

int fcntl(int fd, int op, ...);
